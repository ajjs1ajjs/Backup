# 🔄 NovaBackup - Автономна робота (як Veeam)

## 📋 Огляд

NovaBackup працює як **фонова служба** (аналог Veeam Backup Service):
- ✅ Працює навіть коли GUI закрито
- ✅ Автоматично запускається після старту Windows
- ✅ Виконує заплановані backup jobs
- ✅ Монірить статус backup в реальному часі

## 🛠️ Компоненти

### 1. **NovaBackup.GUI.exe** - Користувацький інтерфейс
- Графічний інтерфейс у стилі Veeam
- Для конфігурації та моніторингу
- Можна закривати - служба продовжує працювати

### 2. **Планувальник завдань (Task Scheduler)** - Фонова служба
- Автоматично встановлюється з MSI
- Запускається при старті Windows
- Виконує заплановані backup jobs
- Перевіряє розклад кожні 5 хвилин

### 3. **SQLite Database** - База даних
- Зберігає всі backup jobs
- Зберігає розклади та налаштування
- Зберігає історію виконання
- Знаходиться в: `%ProgramData%\NovaBackup\backup.db`

## 📦 Встановлення

### Крок 1: Встановити MSI
```cmd
msiexec /i installer\NovaBackup-1.0.0.msi
```

### Крок 2: Перевірити встановлення
```cmd
# Перевірити службу
sc query "NovaBackup Agent"

# Або перевірити планувальник
schtasks /query /fo LIST | findstr NovaBackup
```

### Крок 3: Запустити GUI
```
C:\Program Files\NovaBackup\NovaBackup.GUI.exe
```

## ⚙️ Налаштування автоматичного запуску

### Варіант 1: Через планувальник завдань (рекомендовано)

Після встановлення MSI, планувальник автоматично створює завдання:
- **Назва:** `NovaBackup Agent`
- **Запуск:** При старті системи
- **Користувач:** SYSTEM
- **Дія:** Запускати кожні 5 хвилин

### Варіант 2: Ручне додавання до планувальника

```cmd
schtasks /create /tn "NovaBackup Agent" /tr "\"C:\Program Files\NovaBackup\NovaBackup.Agent.exe\"" /sc onstart /ru SYSTEM
```

### Варіант 3: Через реєстр (автозавантаження)

Додати ключ до реєстру:
```cmd
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "NovaBackup" /t REG_SZ /d "\"C:\Program Files\NovaBackup\NovaBackup.GUI.exe\"" /f
```

## 🎯 Як це працює (як Veeam)

### 1. **Встановлення**
```
MSI → Встановлює файли → Реєструє службу → Створює планувальник
```

### 2. **Старт Windows**
```
Windows Startup → Task Scheduler → NovaBackup Agent → Перевірка розкладу
```

### 3. **Виконання Backup**
```
Plan Time → Agent перевіряє БД → Знаходить активні jobs → Запускає Backup Engine → Виконує backup → Записує результат в БД
```

### 4. **Моніторинг**
```
GUI відкривається → Зчитує БД → Показує статус → Оновлюється в реальному часі
```

## 🔧 Управління службою

### Перевірити статус
```cmd
sc query "NovaBackup Agent"
```

### Запустити службу
```cmd
net start "NovaBackup Agent"
```

### Зупинити службу
```cmd
net stop "NovaBackup Agent"
```

### Перезапустити службу
```cmd
net stop "NovaBackup Agent" && net start "NovaBackup Agent"
```

### Видалити службу
```cmd
sc delete "NovaBackup Agent"
```

## ❓ Усунення проблем

### Служба не запускається
1. Перевірте логи: `Event Viewer → Windows Logs → Application`
2. Перевірте права: служба має запускатися від SYSTEM
3. Перевірте наявність файлів в `C:\Program Files\NovaBackup\`

### Backup не виконується за розкладом
1. Відкрийте Task Scheduler
2. Знайдіть `NovaBackup Agent`
3. Перевірте історію виконання
4. Перевірте БД: чи є активні jobs

### GUI не бачить службу
1. Закрийте GUI
2. Перезапустіть службу: `net stop "NovaBackup Agent" && net start "NovaBackup Agent"`
3. Відкрийте GUI знову
