Система Резервного Копіювання Даних

🚀 Швидкий Старт

Якщо ви вперше користуєтесь системою, виконайте ці кроки:

1 Відкрийте "Швидке Резервне Копіювання" у лівому меню
2 Оберіть тип джерела:
  • 📁 Локальні файли - файли на цьому комп'ютері
  • 🌐 Мережа - файли в мережі (\\server\share)
  • ☁️ Хмара - S3, Azure, Google Cloud
3 Вкажіть шлях до даних (наприклад: C:\Documents або D:/MyFiles)
4 Оберіть сховище призначення (куди зберігати бекап)
5 Налаштуйте стиснення та шифрування за бажанням
6 Натисніть "Запустити Резервне Копіювання"
💡 Порада: Використовуйте forward slashes (D:/Documents) замість backslashes (D:\Documents) - це уникне проблем з Unicode символами!

📋 Створення Завдання Резервного Копіювання

🎯 Типи Завдань

  • Файлове копіювання - резервування файлів і папок
  • Бази даних - MySQL, PostgreSQL, SQLite
  • Віртуальні машини - Hyper-V VM backup
  • Хмарне копіювання - AWS S3, Azure Blob, Google Cloud

⚙️ Налаштування Розкладу

  • Вручну - тільки коли натиснете кнопку
  • Щодня - кожного дня у вказаний час
  • Щотижня - у вибрані дні тижня
  • Cron - власний розклад (наприклад: 0 2 * * * - щодня о 2:00)

💾 Політика Зберігання

  • Днів зберігання - видаляти бекапи старіше N днів
  • Копій зберігання - зберігати максимум N копій

♻️ Відновлення Даних

1 Відкрийте розділ "Відновлення" у лівому меню
2 Оберіть тип відновлення:
  • 📁 Файли і Папки - окремі файли з бекапу
  • 🗄️ Бази Даних - відновлення БД
  • 🖥️ Віртуальні Машини - VM Hyper-V
3 Вкажіть шлях до бекапу або оберіть з історії
4 Виберіть файли для відновлення
5 Вкажіть місце відновлення:
  • Оригінальне місце
  • Нова папка (вкажіть шлях)
6 Натисніть "Почати Відновлення"

📊 Моніторинг та Звіти

📈 Панель Керування

Показує:

  • Кількість активних завдань
  • Останні сесії бекапу
  • Статус системи
  • Швидкі дії

📜 Історія Сесій

Кожна сесія містить:

  • Статус (успішно/помилка)
  • Тривалість
  • Розмір даних
  • Ступінь стиснення
  • Лог операцій

💡 Корисні Поради

✅ Найкращі Практики

  • Зберігайте бекапи на окремому диску або мережевому сховищі
  • Використовуйте стиснення для економії місця
  • Увімкніть шифрування для конфіденційних даних
  • Налаштуйте сповіщення про результати бекапу
  • Періодично перевіряйте відновлення

⚠️ Уникнення Проблем

  • Не використовуйте однакові шляхи для джерела і призначення
  • Переконайтеся, що на диску достатньо місця
  • Не видаляйте бекапи під час виконання завдання
  • Використовуйте коректні шляхи без Unicode символів
⚠️ Увага: При копіюванні шляхів з Windows Explorer можуть з'являтись невидимі Unicode символи. Вводьте шляхи вручну або використовуйте forward slashes: D:/Documents

⌨️ Гарячі Клавіші

  • F5 - Оновити панель керування
  • Ctrl+S - Зберегти налаштування
  • Esc - Закрити модальне вікно

🛠️ Технічна Підтримка

Якщо виникли проблеми:

  1. Перевірте лог файли в розділі "Сесії"
  2. Переконайтеся, що сервер працює (http://localhost:8050)
  3. Перезапустіть службу NovaBackup
  4. Зверніться до адміністратора системи
