Новий план розвитку NovaBackup (2026-03-12)

Мета
- Завершити залишені фази Roadmap та стабілізувати продукт до релізного стану з мінімальною залежністю від зовнішніх сервісів.

Поточний стан (коротко)
- Фаза 1: клієнтська консоль (WPF) частково реалізована; є базова навігація, wizard-і та відображення статусів. 
- Фаза 2: Enterprise Data Movers частково реалізовано (RPC, VMware HotAdd, Hyper-V Data Mover).
- Фаза 3: Guest Processing & Application-Aware Backup частково підготовлена; VSS провайдери в pkg/providers/vss існують.
- Фаза 4: Instant VM Recovery (NFS) реалізовано частково через інфраструктуру recovery.
- Фаза 5: SOBR, Immutable Backups, Snapshots — частково реалізовано (реалізовано ядро, але потребує доведення до стабільного стану).

Що залишилося (розподіл за напрямками)
- Гостьова обробка (GIP): відсутня повна інтеграція з guest-OS, VSS інтеграція потребує повного покриття.
- Фіксація UI: помилки в XAML та MVVM зразками; нестабільні прив'язки даних на деяких екранах (HomeView, JobWizard).
- Фейс ліни процес Wizard: оновити ViewModel та відповідні XAML, синхронізувати з API контрактами (створення кавер-методів CreateJobAsync з деталями).
- Бекенд API: потрібні коректні кінцеві точки для отримання даних та створення задач (поточна реалізація в Go може потребувати доопрацювань відповідно до нових полів).
- GIP: нарешті інтегрувати з VSS, Linux pre/post freeze скрипти та моніторинг.
- Тестування:Unit-тести для провайдерів, інтеграційні тести для UI та API.
- Безпека/резерви: перевірка логів, дедуплікація та збереження даних.

План на 8 тижнів (еволюційний, по тикам)
- Тиждень 1-2: Фіксація UI та стабільність MVVM
  - Виправити помилки в HomeView.xaml та JobWizardWindow.xaml.
  - Стабілізувати JobWizardViewModel: повернутися до базових полів (JobName, JobDescription, AvailableSources, SelectedSource, AvailableRepositories, SelectedRepository, RetentionDays, EnableGuestProcessing, GuestCredentialsId) та синхронізувати з XAML.
- Тиждень 3-4: GIP та VSS інтеграція
  - Розробити загальний інтерфейс для Guest Processing Proxy, додати заглушки/моки для тестів.
  - Розширити pkg/providers/vss для покриття основних кейсів: VSS, AD/Exchange, Linux pre/post freeze.
- Тиждень 5-6: BackendAPI та контракти
  - Вдосконалити Go API: забезпечити потрібні полями контрактів для створення джобів, списків репозиторіїв, та відображення статусів.
  - Адаптувати ApiClient для нових полів та форматов (schedule, retention, guest processing).
- Тиждень 7-8: Тестування та стабілізація
  - Написати юніти-тести для провайдерів та основних сервісів.
  - Виконати інтеграційні тести між WPF-клієнтом та Go-сервісом.
  - Пройти базову демонстрацію: 1) створити джоб, 2) запустити Instant Recovery, 3) переглянути статуси.

Залежності та ризики
- Залежність від стабільності Go API та нових контрактів; потребує частого синхронізування з фронтендом.
- Потрібна стабільна збірка WPF-проекту; можливі конфлікти між MVVM-підходами та UI-елементами.
- Наявність коректних тестів для критичних сценаріїв (GIP, VSS, SOBR).

Наступні кроки після вашого підтвердження
- Я зараз приступлю до фікса UI (HomeView.xaml) та відновлення стабільної версії JobWizardViewModel, після чого реалізую базову інтеграцію CreateJob на API з оновленими полями.
- Після цього додам GIP-заглушки та варіативні сценарії VSS.
