## 📊 1. РОЗШИРЕНІ НАЛАШТУВАННЯ БЕКАПУ

### 🗜️ **1.1 Рівень Стиснення**

**Опис:** Дозволяє вибрати рівень стиснення даних для оптимізації між швидкістю та розміром.

**Опції:**
| Значення | Назва | Використання |
|----------|-------|--------------|
| 0 | Без стиснення (None) | Для вже стиснутих файлів |
| 1 | Швидке (Fast) | Пріоритет швидкості |
| 5 | Нормальне (Normal) | **За замовчуванням**, баланс |
| 9 | Максимальне (Maximum) | Пріоритет розміру |

**Переваги:**
- ⚡ Швидке: на 40-60% швидше, стиснення на 20-30%
- 📦 Нормальне: оптимальний баланс
- 🗜️ Максимальне: до 80% економії місця, але на 50% повільніше

---

### 📦 **1.2 Розмір Блоку (для Дедуплікації)**

**Опис:** Визначає розмір блоків для дедуплікації даних.

**Опції:**
| Розмір | Значення (bytes) | Використання |
|--------|------------------|--------------|
| 256 KB | 262,144 | Для малих файлів |
| 512 KB | 524,288 | Стандартний |
| 1 MB | 1,048,576 | **За замовчуванням**, універсальний |
| 2 MB | 2,097,152 | Для великих файлів |
| 4 MB | 4,194,304 | Максимальна ефективність |

**Переваги:**
- Менші блоки → краща дедуплікація, більше метаданих
- Більші блоки → швидша обробка, менше метаданих

---

### 🔢 **1.3 Кількість Потоків**

**Опис:** Контролює паралелізм операцій бекапу.

**Діапазон:** 1-32 потоків  
**За замовчуванням:** 4 потоки

**Рекомендації:**
| Сценарій | Рекомендовано |
|----------|---------------|
| Одноядерний CPU | 1-2 потоки |
| 2-4 ядра | 4-8 потоків |
| 8+ ядер | 8-16 потоків |
| Сервер | 16-32 потоки |

**Переваги:**
- ⚡ Більше потоків = швидший бекап
- 💾 Менше потоків = менше навантаження на систему

---

### 🚫 **1.4 Виключення (Exclusion Patterns)**

**Опис:** Дозволяє виключити файли/папки з бекапу за шаблонами.

**Приклади:**
```
*.tmp, *.log, ~$*, *.cache
AppData\Local\Temp
Windows\Temp
*.pst, *.ost (для Outlook)
node_modules
.git
```

**Підтримка Wildcard:**
- `*` - будь-яка кількість символів
- `?` - один символ

**Переваги:**
- 📉 Менший розмір бекапу
- ⚡ Швидше виконання
- 🎯 Тільки важливі дані

---

### 📜 **1.5 Скрипти Перед/Після Бекапу**

**Опис:** Автоматизація задач до та після виконання бекапу.

**Pre-Before Script (Перед бекапом):**
- Зупинка баз даних
- Закриття програм
- Створення snapshot
- Очищення тимчасових файлів

**Post-After Script (Після бекапу):**
- Запуск баз даних
- Надсилання звітів
- Синхронізація з хмарою
- Очищення старих бекапів

**Приклади:**
```batch
:: Pre-backup: Stop MySQL
net stop MySQL80

:: Post-backup: Send notification
curl -X POST https://hooks.slack.com/... -d "Backup completed!"
```

---

### 🔄 **1.6 Інкрементальне Бекапування**

**Опис:** Зберігає тільки зміни з моменту останнього бекапу.

**Опції:**
- ☑️ Інкрементальне бекапування
- 📅 Повний бекап кожні N днів (1-30)

**Переваги:**
- 📉 На 90% менший розмір щоденних бекапів
- ⚡ На 80% швидше виконання
- 💰 Економія місця в сховищі

**Стратегія:**
```
День 1: Повний бекап (100 GB)
День 2-7: Інкрементальні (2-5 GB кожен)
День 8: Повний бекап (100 GB)
```
