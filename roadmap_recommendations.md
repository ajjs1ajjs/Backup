# Рекомендації до roadmap.md

## 1. Додати технічний стек (на початку документу)

Рекомендовано визначити:
- **Мова сервера**: .NET 8 / C#
- **Мова агентів**: C++ (C++20)
- **База даних**: PostgreSQL
- **Гіпервізори**: VMware (пріоритет) → Hyper-V → KVM
- **Протокол**: gRPC + Protocol Buffers
- **ОС агентів**: Windows Server 2019+ / Linux (Ubuntu 22.04+)

## 2. Виправити залежності між фазами

| Проблема | Рішення |
|----------|---------|
| Фаза 2.2 залежить від гіпервізора | Додати вибір VDDK/VixDiskLib перед початком Фази 2 |
| Фаза 4.2 залежить від ОС | Визначити цільові ОС на етапі Фази 1 |

## 3. Додати відсутні розділи

### 3.1. Security
- Шифрування даних at rest (AES-256)
- TLS для gRPC
- Аутентифікація агентів (сертифікати)

### 3.2. Network
- Вибір протоколу: TCP / RDMA (для high-speed)
- Підтримка bandwidth throttling

### 3.3. Monitoring
- Health checks для агентів
- Логування (ELK stack / Seq)

## 4. Корекція термінів

| Фаза | Поточний термін | Рекомендовано |

Обґрунтування: Enterprise-рішення потребують більше часу на стабілізацію.

## 6. Пріоритизація фічей

### MVP (обов'язково)
- Повний бекап VM
- Відновлення VM
- REST API
- Базова конфігурація

### Post-MVP
- CBT (інкрементний бекап)
- Fast Clone
- FLR (File-Level Recovery)
- Multi-VM паралельний бекап
- GraphQL API (опціонально)
